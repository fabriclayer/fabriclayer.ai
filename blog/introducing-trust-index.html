Introducing the Fabric Trust Index

Open-source trust scores for 5,800+ AI services. Six signals, one score, fully automated. The safety layer the agent economy has been missing.

Today we're launching the Fabric Trust Index — a free, public scoring engine that rates every AI service, SDK, agent tool, and MCP skill on a scale of 0.00 to 5.00. No account required. No paywalls. Just search a service and get an instant trust score backed by real data.

We built Fabric because the AI ecosystem has a trust problem. Thousands of AI tools launch every week. Anyone can build one. Most are never reviewed for safety, privacy, or reliability. A slick landing page doesn't mean a tool is safe, and a tool that was safe last month can push a bad update, change its privacy policy, or get acquired by someone with different intentions — and nobody tells you.

Until now, there was no universal way to check.

Six signals. One score.

Every AI service in the index receives a composite trust score computed from six independently weighted signals. Each signal scores 0–5 individually, then is multiplied by its weight. The six weighted contributions sum to the final score.

Vulnerability & Safety

Transparency

Publisher Trust

Who built this? Account age, organisational membership, cross-registry identity consistency, maintained package count, and prior security incidents.

Why six signals? Any single metric can be gamed. Download counts can be inflated. GitHub stars can be bought. Even vulnerability databases have blind spots. By combining six independent signals across security, operations, maintenance, adoption, transparency, and publisher identity — an attacker would need to simultaneously spoof all of them to pass. That's the point.

Three tiers. One decision field.

Every scored service falls into one of three tiers. For AI agents, we reduce it to a single decision field — allow, review, or block — so an agent can make a routing decision without parsing scores.

ScoreStatusWhat it means
3.25 – 5.00 Trusted Low risk across all signals. Agents can auto-approve.
1.00 – 3.24 Caution Some signals flagged. Require human confirmation.
0.00 – 0.99 Blocked Significant risks detected. Deny by default.

Supply chain protection

The most dangerous attack vector in the AI ecosystem isn't a suspicious new tool — it's a trusted one that changes hands. A repo transfer is the most common vector for supply chain attacks on packages. The attacker transfers a popular repo to their account, publishes a new version with malicious code, and the package manager trusts it because the repo looks the same.

Fabric detects this. Our override system evaluates eight hard safety rules after every scoring cycle, including three designed specifically for supply chain integrity:

  • Archived repositories — if a GitHub repo is archived, the service is blocked at 0.99 regardless of other signals. The maintainer has walked away.
  • Deprecated packages — npm packages with the deprecated flag are blocked. The maintainer is explicitly telling you to stop using it.
  • Ownership transfers — when a repo changes hands, Fabric distinguishes between a same-owner rename (no penalty) and a different-owner transfer (score frozen, –1.0 penalty, flagged for manual review). Without this, the scoring engine would see better metrics on the new owner's repo and increase the trust score — the exact opposite of the correct response.

These overrides are hard caps. No combination of high signal scores can bypass them.

CVE detection in under five minutes

Vulnerability data is the most time-sensitive signal. Fabric runs a two-tier CVE scanning system: a full scan every hour with immediate rescore when a score drops more than 0.5, and a fast-path scanner every five minutes that checks OSV.dev delta feeds for new disclosures. Worst case from CVE publication to score update: five minutes.

CVEs are classified by severity (critical, high, medium, low) and patch status (unpatched, patch available, patched). A critical unpatched CVE is penalised 20× more than a low patched one. The collector traces the full dependency tree — not just direct dependencies — because a package with zero direct vulnerabilities can still score poorly if something three levels deep has a critical CVE.

Fully automated. No manual reviews.

Every score in the Trust Index is computed from real-time data pulled from public APIs. No human reviewers, no pay-to-play, no editorial bias. A provider doesn't need to opt in — or even know they're being scored — for Fabric to evaluate them.

The only way to improve a score is to improve the underlying signals. Fix vulnerabilities. Maintain the project. Improve uptime. Verify your identity. Publish documentation. There is no other path.

Data flows in from 10 sources: OSV.dev for vulnerability data, GitHub API for maintenance and transparency, npm and PyPI registries for adoption, VirusTotal for malware scanning, ClawHub for AI skill analysis, HTTP pings for operational health, and Claude for AI-generated trust assessments that explain scores in plain English.

Eight discovery sources

The Trust Index currently covers 5,800+ AI services, and grows daily through eight automated discovery sources. Registry crawlers search npm and PyPI by AI-relevant keywords. A news scanner checks GitHub Trending, Product Hunt, YC Launches, and Show HN for day-one catches. ClawHub skills and MCP servers are crawled separately. A curated watchlist catches platform-only products like Cursor, Perplexity, and Bolt.new that have no package to find in a registry.

A new AI tool launched on Product Hunt at 9am can be discovered, enriched with metadata, scored across all six signals, assessed, and serving an embeddable trust badge — all by the next morning. No human intervention at any step.

What's next

The Trust Index is the foundation — the scoring engine everything else builds on. Coming next:

  • Publisher profiles — claim your service, verify your identity, improve your transparency score.
  • Custom weights — rebalance the scoring formula to match your priorities without changing the underlying data.
  • MCP integration — tools that let agents discover, evaluate, and route to services with trust checks in the decision loop.
  • Public API — programmatic access to trust scores so agents can check any service before granting it access.

The full scoring methodology, data sources, and technical architecture are documented at fabriclayer.ai/docs. Everything is transparent because trust infrastructure should be.

The Trust Index is free, public, and requires no account. Search any AI service and get an instant trust score.
